# Dynamic Mode Guidelines

## 触发条件

仅在以下条件满足时才进入动态模式：

1. 静态审计 `top_confidence=unknown`。
2. 或用户明确要求启用动态模式。
3. 且总控启用了 `--allow-dynamic`。

## 成本优先策略

1. 默认 `budget-level=low`，先走最省成本路径。
2. 动态阶段必须先跑 Hook 采集。
3. 仅在证据不足、Hook 失败或用户强制时升级到 CDP 断点。
4. 通过时间窗、事件上限、字段截断控制日志体积。

## 动态模式允许做的事

1. 抓取运行时动态加载的 JS URL。
2. 捕获 Hook 日志（`fetch/xhr/CryptoJS/subtle`）和控制台关键输出。
3. 通过 CDP URL/XHR 断点暂停，记录暂停点调用栈快照后自动恢复。

## 动态模式禁止做的事

1. 不做无关页面操作。
2. 不做破坏性测试。
3. 不修改系统或全局环境。

## 执行顺序

1. 先静态，再动态。
2. 动态按阶梯执行：`Hook -> (必要时) CDP`。
3. 动态只补证据，不替代静态证据链。
4. 动态失败即回到手动保存兜底。
